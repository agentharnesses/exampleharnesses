## Description Format

```
[2-3 sentence hook — the core question or problem the video answers]

[1 sentence on what the viewer will understand by the end]

Chapters:
00:00 Introduction
[remaining chapters]

Resources:
- [Link to code repo or notebook, if applicable]
- [Link to relevant paper or article, if applicable]
- [Link to related video or article, if applicable]

[Subscribe CTA — one line, e.g. "Subscribe for more deep dives on ML and software engineering."]
```

## Chapter Markers

Format: `MM:SS Title`

Add chapters whenever the video shifts to a meaningfully different topic or step. Aim for 5-15 chapters on a typical video. The first chapter is always `00:00 Introduction`. Chapter titles should be descriptive enough to be useful out of context (someone skimming the chapter list should be able to find what they want).

## Tags

Use 10-15 tags. Mix: 2-3 broad tags (e.g. `programming`, `machine learning`, `software engineering`) + 5-8 specific topic tags (e.g. `pytorch`, `attention mechanism`, `python tutorial`) + 2-3 format/audience tags (e.g. `deep dive`, `explained`, `for developers`).

## Thumbnail Brief Format

- **Title text:** The short punchy phrase overlaid on the thumbnail (max ~5 words, high contrast)
- **Visual concept:** What the image shows — diagram, code snippet, photo, abstract graphic
- **Style notes:** Color palette, mood, whether to include a face/headshot
- **Example:** "Title: 'Attention, Explained' | Visual: transformer diagram with attention weights highlighted in orange | Style: dark background, technical aesthetic"
