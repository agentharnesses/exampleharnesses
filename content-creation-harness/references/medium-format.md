## Structure

1. **Hook paragraph** — the problem, question, or surprising observation that motivates the article. No background yet.
2. **Context / motivation** — why this matters, what gap it fills, what the reader will walk away knowing
3. **Main content** — broken into H2 sections, each covering one concept or step. H3 for subsections when needed.
4. **Conclusion** — one concrete takeaway and a natural next step or open question. Does not restate the article.

## Formatting

- **Headers:** H2 for main sections, H3 for subsections. H1 is set by Medium as the article title.
- **Code blocks:** Always fenced with a language tag (`python`, `bash`, `javascript`, etc.). Use inline code (backticks) for short variable names, commands, or file paths within prose.
- **Images:** Descriptive alt text that conveys meaning, not just "screenshot." Add a caption when the image needs context that isn't obvious.
- **Callouts:** Use blockquotes (`>`) sparingly for key definitions or warnings worth emphasizing.

## Tags

Use all 5 available tags. Mix: 1-2 broad audience tags (e.g. `Programming`, `Machine Learning`, `Software Engineering`) + 2-3 specific topic tags (e.g. `Python`, `Transformers`, `Distributed Systems`). Specific tags drive discovery; broad tags boost reach.

## Publication Checklist

- [ ] Title and subtitle set
- [ ] Cover image with alt text
- [ ] Tags added (max 5)
- [ ] Canonical URL set if cross-posting
